MVP- 핵심 기능 구현

STEP 1: 데이터베이스 및 기본 환경 설정

  • 회원정보에 일반/전용회원 구분과 유통사코드 입력
  • 유통그룹에 배송가능지역 기능
  • 와일드카드 DNS 설정: `*.orderz.kr` 주소 사용을 위한 DNS 설정

STEP 2: 백엔드 개발 (Back-end)

  • 상품 목록 API 개발 (사용자 유형 및 지역에 따라 동적 데이터 반환)

STEP 3: 프론트엔드 개발 (Front-end)

  • 상품 목록 API 호출 로직 구현
  • API 응답 데이터를 화면에 표시하는 UI 컴포넌트 개발

기능 확장 - 전용몰 경험 강화

MVP 기능 위에 유통사별 브랜딩 기능을 추가하여, '전용 회원'에게 완전한 맞춤형 몰 경험을 제공합니다.

STEP 1: 인프라 설정 및 관리자 기능

  • 로고/GNB 관리: 유통사가 전용 로고와 GNB 메뉴를 설정하는 관리자 기능 구현

STEP 2: 동적 UI 적용

  • 상품 API 응답에 '로고 이미지 주소', 'GNB(상단 메뉴) 정보' 등 추가 UI 데이터를 포함하도록 백엔드 수정
  • 전용 회원이거나 전용 서브도메인 접속 시, API 데이터를 받아 헤더 로고와 GNB 동적 변경

장기 로드맵 (Long-term Roadmap)

핵심 기능 구현 이후, 플랫폼의 경쟁력 강화를 위해 추진할 주요 개발 항목입니다.

B2B 기능 고도화

  • 여신(외상) 시스템 도입: 안정적인 거래 실적을 보유한 회원에게 외상 거래 한도를 부여하여 유연한 자금 운영을 지원합니다.
  • 계약 단가 적용: 전용 회원별 계약 단가를 시스템에 등록하고, 주문 시 자동으로 적용하는 기능을 구현합니다.

고객 UI/UX 개발

  • 상세 장바구니 기능: 유통사별 배송비, 예상 도착일 등을 표시하고 상품 수량을 쉽게 변경하는 기능을 제공합니다.
  • 마이페이지 기능: 주문 내역 조회, 배송 상태 추적, 회원 정보 수정, 결제 수단 관리 등 통합 고객 관리 기능을 구현합니다.
  • 빠른 주문(찜 리스트) 기능 강화: 자주 주문하는 상품 목록에서 수량만 입력하여 바로 주문하는 기능을 고도화합니다.

주요 사용자 경험(UX) 흐름 설계

고객에게 직접적으로 노출되는 핵심적인 프론트엔드 기능 및 인터페이스 흐름입니다.

회원 유형별 주문 흐름

주문 속도와 정확성을 모두 고려한 맞춤형 경험을 제공합니다.

  • 일반회원: 빠른주문 → 장바구니 (유통사별 그룹핑) → 주문서
  • 전용 회원: 빠른주문 → 주문서 (장바구니 생략)
