마케팅 링크 기능 개발 명세서

유통 그룹별 타겟 마케팅 링크 생성 기능 기획안 (개발자 전달용)

1. [관리자] '랜딩 페이지' 관리 (신규 개발)

'봄맞이 세일', '신규회원 이벤트' 등 마케팅에 사용할 랜딩 페이지 목록을 등록하고 관리하는 신규 메뉴를 개발합니다.

주요 기능

  • 랜딩 페이지 등록/수정/삭제 (CRUD)
  • 입력 항목: 페이지 이름, 페이지 URL (예: '/event/spring_sale')
  • 랜딩 페이지 노출 여부(상태) 관리 (예: 사용중, 보류중)

개발 항목

  • DB 테이블 신규 생성 (예: `landing_pages`, status 컬럼 추가)
  • 관리자 API 개발 (CRUD 4종 + 상태 변경 API)
  • 관리자 프론트엔드 페이지 개발 (목록, 등록/수정 폼)

2. [관리자] '유통 그룹 관리' 기능 수정 (기존 페이지 수정)

기존 '유통 그룹 관리' 목록 페이지에 '마케팅 링크' 버튼 및 팝업(모달) 기능을 개발합니다.

주요 기능

  • '마케팅 링크' 버튼 클릭 시 팝업 노출
  • 팝업 내 '랜딩 페이지' 목록 드롭다운으로 선택 (이때 '사용중' 상태인 페이지만 노출)
  • 선택된 페이지와 그룹 코드를 조합하여 최종 링크 생성
  • '복사' 버튼으로 생성된 링크를 클립보드에 복사

개발 항목

  • 프론트엔드 UI/UX 개발 (버튼 추가, 팝업(모달) UI, JS 로직)
  • 랜딩 페이지 목록을 불러오는 API 1종 (팝업 내 드롭다운용, '사용중' 상태 필터링)

3. [쇼핑몰] '회원가입' 로직 수정 (기존 기능 수정)

마케팅 링크를 통해 들어온 방문자가 회원가입 시, 해당 유통 그룹 코드가 자동으로 회원 정보에 저장되도록 수정합니다.

주요 기능

  • 회원가입 페이지 진입 시 URL의 `group` 파라미터 인식
  • 가입 완료 시, 해당 `group` 코드를 신규 회원 DB에 저장

개발 항목

  • 회원가입 페이지 프론트엔드 로직 수정 (파라미터 값 읽기)
  • 회원가입 API 백엔드 로직 수정 (파라미터 값 수신 및 DB 저장)

4. [쇼핑몰] '랜딩 페이지' 개발 (신규 개발)

마케팅 링크의 목적지가 되는 실제 랜딩 페이지(들)를 개발합니다.

주요 기능

  • URL의 `group` 파라미터를 인식
  • 해당 `group` 코드에 속한 **전용 상품만** 페이지에 노출
  • 페이지 내 '회원가입' 버튼/링크에 `?group=...` 파라미터 전달

개발 항목

  • 프론트엔드 페이지 신규 개발 (HTML/CSS/JS)
  • `group` 코드에 맞는 상품 목록을 조회하는 백엔드 API
